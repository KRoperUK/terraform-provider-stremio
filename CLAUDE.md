# CLAUDE.md

Guidance for AI coding agents (Claude Code and others) working in this repository.

## What this is

A Terraform provider for [Stremio](https://www.stremio.com/), built on the
[terraform-plugin-framework](https://github.com/hashicorp/terraform-plugin-framework).
It manages Stremio accounts and synchronises their add-on collections, and exposes
read-only data about installed add-ons and watch history.

- Module: `github.com/kroperuk/terraform-provider-stremio`
- Registry: `KRoperUK/stremio` (`registry.terraform.io/providers/KRoperUK/stremio`)
- Go 1.25+, Terraform 1.6+, plugin protocol 6.0

## Layout

```
main.go                                  # provider entrypoint (providerserver.Serve)
internal/provider/
  provider.go                            # provider schema + Resources()/DataSources() wiring
  client.go                              # Stremio RPC client (POST /api/<method>) + helpers
  client_test.go                         # httptest-based unit tests for the client
  account_resource.go                    # stremio_account resource (CRUD + import)
  installed_addons_data_source.go        # stremio_installed_addons data source
  watch_history_data_source.go           # stremio_watch_history data source
  continue_watching_data_source.go       # stremio_continue_watching data source
  models.go                              # shared tfsdk model types
docs/                                    # generated by tfplugindocs (committed); docs/wiki/ is hand-written
examples/                                # basic / multi-account / local usage examples
scripts/                                 # local dev helpers (plan/apply against examples)
```

### Surface area

- Resource: `stremio_account` — creates/imports an account by `email`/`password`.
  Add-on collections are managed via its optional `transport_urls` set (there is **no**
  separate `stremio_addon_collection` resource).
- Data sources: `stremio_installed_addons`, `stremio_watch_history`,
  `stremio_continue_watching`.

The Stremio API is an RPC-style JSON API: `POST /api/<method>` with `authKey` in the
body, responses wrapped in a `{ "result": ..., "error": ... }` envelope. See `client.go`.

## Conventions — follow these

- **Conventional Commits.** Releases are automated by release-please. Use
  `feat:`, `fix:`, `docs:`, `test:`, `ci:`, `build:`, `chore:`, etc. `feat:`/`fix:`
  drive version bumps; the rest do not.
- **Commits must be GPG-signed.** A repository ruleset requires verified signatures on
  all branches. Ensure `commit.gpgsign=true` and a signing key registered on GitHub.
- **Regenerate and commit docs** whenever you change provider/resource/data-source
  schemas or descriptions. CI fails if `docs/` (excluding `docs/wiki/`) is out of date.
  Keep schema `Description`/`MarkdownDescription` strings accurate — the registry docs
  are generated from them.
- **Keep the schema the source of truth.** When you add or change an attribute, update
  the Go schema, regenerate docs, and update the README/examples to match. Drift between
  README and the actual schema is a bug.
- **Tests:** add/extend unit tests in `internal/provider`. The client is tested with
  `net/http/httptest` — no live Stremio calls. Don't introduce test calls to the real API.
- **No secrets in the repo:** never commit `.env`, `*.tfvars`, `*.tfstate`, or real
  credentials. Credentials flow via env/`TF_VAR_*` (see `scripts/`).
- Run `gofmt`/`golangci-lint` before committing; `terraform fmt` for example `.tf` files
  (enforced by pre-commit).

## CI gates (must pass to merge)

The `main` branch requires these checks: `pre-commit`, `go`, `terraform-docs`.
The `go` job runs: `go mod tidy` (must be clean), build, vet, `go test -race`,
`golangci-lint`, and `govulncheck`. Dependencies are kept current by Dependabot
(`.github/dependabot.yml`).

## Release flow

Push conventional commits to `main` → release-please opens/updates a release PR →
merging it bumps the manifest and creates a `v*` tag → the tag triggers GoReleaser,
which publishes GPG-signed, registry-compatible artifacts. See
[.github/workflows/release.yml](.github/workflows/release.yml).
